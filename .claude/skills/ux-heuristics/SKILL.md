# UX Heuristics Evaluation Guide

## Nielsen의 10가지 사용성 휴리스틱

### H1: 시스템 상태의 가시성 (Visibility of System Status)

사용자에게 현재 상태를 적절한 피드백으로 알려야 합니다.

**캘린더 프로젝트 적용**:
- 현재 선택된 월/연도가 명확히 표시되는가?
- 일지가 있는 날짜와 없는 날짜의 구분이 명확한가?
- 로딩 중/데이터 없음 상태가 표시되는가?
- 🔥 이모지의 의미를 사용자가 이해할 수 있는가?

### H2: 시스템과 현실 세계의 일치 (Match Between System and Real World)

사용자 언어와 관습을 따라야 합니다.

**캘린더 프로젝트 적용**:
- 요일 표시가 사용자의 문화권에 맞는가? (일~토 vs 월~일)
- 날짜 포맷이 자연스러운가?
- 섹션 제목이 직관적인가? (9 to 6, 6시 이후 등)

### H3: 사용자 제어와 자유 (User Control and Freedom)

사용자가 실수로 진입한 상태에서 쉽게 빠져나올 수 있어야 합니다.

**캘린더 프로젝트 적용**:
- Sheet(일지 상세)를 쉽게 닫을 수 있는가? (X 버튼, Escape, 바깥 클릭)
- 이전 월로 쉽게 돌아갈 수 있는가?
- 오늘 날짜로 빠르게 이동할 수 있는가?

### H4: 일관성과 표준 (Consistency and Standards)

같은 단어/행동/상황은 같은 의미여야 합니다.

**캘린더 프로젝트 적용**:
- 모든 DayCell이 동일한 인터랙션 패턴을 따르는가?
- 네비게이션 패턴이 일관적인가? (버튼 + 키보드)
- 테마 색상이 월 전체에서 일관되게 적용되는가?

### H5: 오류 예방 (Error Prevention)

오류가 발생하기 전에 예방해야 합니다.

**캘린더 프로젝트 적용**:
- 일지가 없는 날짜 클릭 시 빈 Sheet가 열리지 않는가?
- 존재하지 않는 월/연도로 이동이 방지되는가?
- 데이터 파싱 실패 시 적절한 폴백이 있는가?

### H6: 인지보다 인식 (Recognition Rather Than Recall)

사용자가 기억에 의존하지 않도록 정보를 제공합니다.

**캘린더 프로젝트 적용**:
- 각 날짜 셀에 요약이 표시되어 클릭 전에 내용을 짐작할 수 있는가?
- 현재 월/연도가 항상 보이는가?
- 네비게이션 버튼에 레이블이 있는가?

### H7: 유연성과 효율성 (Flexibility and Efficiency of Use)

숙련 사용자를 위한 단축키를 제공합니다.

**캘린더 프로젝트 적용**:
- 키보드 네비게이션(←/→)이 지원되는가?
- 특정 월로 바로 이동할 수 있는가?
- 터치 제스처(스와이프)가 지원되는가?

### H8: 미학적이고 최소한의 디자인 (Aesthetic and Minimalist Design)

불필요한 정보를 제거합니다.

**캘린더 프로젝트 적용**:
- 각 셀에 표시되는 정보량이 적절한가? (과도 vs 부족)
- 시각적 잡음이 최소화되었는가?
- 중요한 정보(날짜, 이모지, 요약)가 우선적으로 보이는가?

### H9: 오류 인식, 진단, 복구 (Help Users Recognize, Diagnose, and Recover from Errors)

오류 메시지는 문제를 명확히 설명하고 해결책을 제시합니다.

**캘린더 프로젝트 적용**:
- 일지 파일 로드 실패 시 사용자에게 명확한 메시지가 표시되는가?
- 빈 월(일지 없음)에 대한 안내가 있는가?

### H10: 도움말과 문서 (Help and Documentation)

필요할 때 쉽게 찾을 수 있는 도움말을 제공합니다.

**캘린더 프로젝트 적용**:
- 🔥 이모지의 의미에 대한 설명(tooltip 등)이 있는가?
- 키보드 단축키 안내가 있는가?

## 접근성 체크리스트 (WCAG 2.1 AA)

### 인지 가능 (Perceivable)

- [ ] 모든 비텍스트 콘텐츠에 대체 텍스트 (alt, aria-label)
- [ ] 색상만으로 정보를 전달하지 않음
- [ ] 텍스트 대비비 4.5:1 이상
- [ ] 텍스트 크기 조절 시 레이아웃 깨지지 않음

### 조작 가능 (Operable)

- [ ] 모든 기능이 키보드로 접근 가능
- [ ] 키보드 트랩(빠져나갈 수 없는 상태) 없음
- [ ] 포커스 순서가 논리적
- [ ] 포커스 인디케이터가 시각적으로 명확
- [ ] 터치 대상 크기 최소 44x44px

### 이해 가능 (Understandable)

- [ ] 페이지 언어가 명시됨 (lang 속성)
- [ ] 일관된 네비게이션 패턴
- [ ] 일관된 컴포넌트 식별 방법
- [ ] 입력 오류 시 명확한 설명

### 견고함 (Robust)

- [ ] 유효한 HTML 마크업
- [ ] ARIA 속성이 올바르게 사용됨
- [ ] 커스텀 컴포넌트의 이름, 역할, 값이 프로그래밍적으로 결정 가능

## 사용자 여정 분석 포인트

### 핵심 사용자 여정

```
1. 랜딩 → 현재 월 캘린더 확인
2. 월 네비게이션 → 원하는 월로 이동
3. 날짜 선택 → 일지 상세 확인
4. 일지 읽기 → Sheet에서 섹션별 내용 확인
5. Sheet 닫기 → 캘린더로 복귀
```

### 각 단계별 평가 기준

- **효율성**: 최소 클릭/탭으로 목표 달성 가능한가?
- **학습 용이성**: 처음 방문한 사용자가 바로 사용 가능한가?
- **만족도**: 시각적으로 쾌적하고 반응이 즉각적인가?
- **오류 방지**: 잘못된 조작으로 인한 혼란이 없는가?
